# 个人作品集展示网站 — 设计说明

日期：2026-06-03 | 状态：已确认

## 概述

基于 `对话.txt` 模板，搭建一个个人数据分析作品集展示网站。采用**纯静态 + JS 数据驱动**方案，单页滚动结构，橙柚暖色配色，个人信息暂用占位。

## 文件结构

```
项目展示/
├── index.html          # 页面骨架（纯 HTML 框架，不含项目卡片）
├── style.css           # 全部样式（橙柚配色、响应式、动画）
├── script.js           # 项目数据数组 + 卡片渲染逻辑 + 平滑滚动
└── images/             # 项目截图文件夹
```

## 技术决策

- **方案三：纯静态 + JS 数据驱动**。一个 HTML + 一个 CSS + 一个 JS，无构建工具，无后端，双击 `index.html` 即可预览，可直接部署到 GitHub Pages。
- 项目信息存储在 `script.js` 顶部的 `projects` 数组中，页面加载时 JS 自动遍历数组生成卡片 DOM。

## 配色

| 用途 | 颜色 |
|------|------|
| Hero 渐变 | `#fbbf24` → `#f97316` → `#f4726a` |
| 技术标签背景 | `#fff3e0` |
| 技术标签文字 | `#e65100` |
| 按钮/链接强调 | `#f97316` |
| 成果高亮背景 | 保持绿色 `#e6fffa`（与暖色不冲突） |
| 页脚背景 | 保持深色 `#1a1a2e` |

## 页面结构

单页滚动，从上到下：
1. **导航栏** — 粘性置顶，Logo + 三个锚点链接（项目/关于/联系）
2. **Hero 区** — 橙柚三色渐变背景，姓名 + 一句话简介 + 跳转按钮
3. **项目展示区** — JS 动态渲染的卡片列表，每张卡片含截图、描述、技术标签、功能列表、成果
4. **关于我区** — 白底居中文字
5. **页脚** — 深色背景，联系方式链接 + 版权

## 数据模型（script.js projects 数组）

每个项目对象字段：

```javascript
{
  name: string,       // 项目名称
  desc: string,       // 简短描述
  tech: string[],     // 技术标签，如 ['Python','Pandas','Plotly','Flask']
  features: string[], // 核心功能列表，如 ['数据清洗','RFM模型','销售预测']
  result: string,     // 项目成果（可选，为空则隐藏该区域）
  img: string,        // 截图路径，如 'images/dashboard.png'
  demo: string,       // 在线演示链接（可选，为空则隐藏按钮）
  repo: string        // GitHub 仓库链接（可选）
}
```

## 初始数据

初始展示本项目本身（即作品集网站），个人信息使用"张三"占位：

- 项目名：个人作品集展示网站
- 技术栈：HTML、CSS、JavaScript
- 功能：橙柚暖色渐变主题、JS 数据驱动卡片渲染、响应式布局、平滑滚动导航
- 成果：纯静态文件，双击即预览，可直接部署到 GitHub Pages

## 响应式

保持原模板的响应式断点（768px），移动端隐藏导航链接，缩小标题字号。

## 添加新项目的方式

1. 截图放入 `images/` 文件夹
2. 在 `script.js` 的 `projects` 数组末尾追加一个对象
3. 刷新浏览器即可看到新卡片

不需要修改 HTML 或 CSS。

## 验证方式

1. 双击 `index.html` 用浏览器打开
2. 检查：排版正常、配色为橙柚渐变、项目卡片内容完整
3. 检查：导航点击平滑滚动到对应区域
4. 在 `script.js` 的 `projects` 数组中追加一条数据，刷新确认新卡片自动出现
5. 缩窄浏览器窗口到 768px 以下，确认移动端布局正常
